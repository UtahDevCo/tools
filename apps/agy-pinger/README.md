# Antigravity (`agy`) Session Pinger for NixOS

This utility directory contains a robust, lightweight bash script (`agy-ping.sh`) designed to automate pinging the Antigravity `agy` CLI at specific times every morning (**4:00 AM** and **9:05 AM**) to keep your developer sessions running or pre-start them cleanly.

---

## What the Script Does

The [agy-ping.sh](file:///home/chris/dev/quiver-hq/projects/tools/apps/agy-pinger/agy-ping.sh) script:
1. Standardizes variables and paths (using absolute executable path `/etc/profiles/per-user/chris/bin/agy`).
2. Changes working directory to the project root `/home/chris/dev/quiver-hq/projects/tools`.
3. Triggers a quick non-interactive prompt `agy --print "System ping"` to start or maintain the Antigravity session.
4. Appends clean logs with timestamps and outputs to `/home/chris/.gemini/antigravity/agy-pinger.log` for debugging and history.

## NixOS Integration Guides

Because NixOS configuration is declarative, scheduling must be registered in your system profile or user home profile. Select the option below that best fits your workspace.

### Option 1: Native Systemd Timers (NixOS System Level) - *Recommended*
This is the most standard, robust NixOS configuration. It handles logging through `systemd-journald` natively and catches up on missed triggers (e.g., if the laptop was sleeping) automatically.

Add the following to `/etc/nixos/configuration.nix` (or your imported NixOS modules):

```nix
# /etc/nixos/configuration.nix
systemd.services.agy-pinger = {
  description = "Ping Antigravity agy CLI to start/keep-alive a session";
  serviceConfig = {
    Type = "oneshot";
    User = "chris";
    WorkingDirectory = "/home/chris/dev/quiver-hq/projects/tools";
    ExecStart = "/home/chris/dev/quiver-hq/projects/tools/apps/agy-pinger/agy-ping.sh";
    Environment = [
      "HOME=/home/chris"
      "USER=chris"
      "PATH=/run/current-system/sw/bin:/etc/profiles/per-user/chris/bin:/home/chris/.nix-profile/bin"
    ];
  };
};

systemd.timers.agy-pinger = {
  description = "Trigger Antigravity agy pinger daily at 4:00 AM and 9:05 AM";
  timerConfig = {
    OnCalendar = [ "*-*-* 04:00:00" "*-*-* 09:05:00" ];
    Unit = "agy-pinger.service";
    Persistent = true; # Run missed triggers if system was offline/asleep
  };
  wantedBy = [ "timers.target" ];
};
```

Apply the changes:
```bash
sudo nixos-rebuild switch
```

### Option 2: User-level Systemd Timers (Home Manager) - *Very Clean*
If you manage your configuration using **Home Manager**, you can deploy this completely in user space without requiring root privileges or system configuration edits.

Add the following to your Home Manager config (e.g., `home.nix` or imported module):

```nix
# home.nix
systemd.user.services.agy-pinger = {
  Unit = {
    Description = "Ping Antigravity agy CLI to start/keep-alive a session";
  };
  Service = {
    Type = "oneshot";
    WorkingDirectory = "/home/chris/dev/quiver-hq/projects/tools";
    ExecStart = "/home/chris/dev/quiver-hq/projects/tools/apps/agy-pinger/agy-ping.sh";
    Environment = [
      "HOME=%h"
      "PATH=/run/current-system/sw/bin:/etc/profiles/per-user/chris/bin:%h/.nix-profile/bin"
    ];
  };
};

systemd.user.timers.agy-pinger = {
  Unit = {
    Description = "Trigger Antigravity agy pinger daily at 4:00 AM and 9:05 AM";
  };
  Timer = {
    OnCalendar = [ "*-*-* 04:00:00" "*-*-* 09:05:00" ];
    Unit = "agy-pinger.service";
    Persistent = true;
  };
  Install = {
    WantedBy = [ "timers.target" ];
  };
};
```

Apply the changes:
```bash
home-manager switch
```

### Option 3: Legacy Cron Jobs
If you explicitly prefer a classic crontab, add the declarative cron service configuration to your system file:

```nix
# /etc/nixos/configuration.nix
services.cron = {
  enable = true;
  systemCronJobs = [
    # Minute Hour DayOfMonth Month DayOfWeek User Command
    "0 4 * * *   chris   /home/chris/dev/quiver-hq/projects/tools/apps/agy-pinger/agy-ping.sh"
    "5 9 * * *   chris   /home/chris/dev/quiver-hq/projects/tools/apps/agy-pinger/agy-ping.sh"
  ];
};
```

Apply the changes:
```bash
sudo nixos-rebuild switch
```

## Operations & Monitoring Commands

### Testing the Script Manually
You can test the pinger script directly:
```bash
/home/chris/dev/quiver-hq/projects/tools/apps/agy-pinger/agy-ping.sh
```

### Checking the Pinger Logs
To view the persistent text log generated by the script:
```bash
tail -f /home/chris/.gemini/antigravity/agy-pinger.log
```

### Checking Systemd Services (If using Option 1 or 2)
If you are using **System-wide Systemd Timers**:
```bash
# Check timer status and next scheduled run
systemctl status agy-pinger.timer

# View execution logs of the pinger service
journalctl -u agy-pinger.service -f

# Trigger the service manually via systemd to verify it works
sudo systemctl start agy-pinger.service
```

If you are using **Home Manager Systemd Timers** (as a normal user, no `sudo` required):
```bash
# Check user timer status and next scheduled run
systemctl --user status agy-pinger.timer

# View user service logs
journalctl --user -u agy-pinger.service -f

# Trigger the user service manually
systemctl --user start agy-pinger.service
```
